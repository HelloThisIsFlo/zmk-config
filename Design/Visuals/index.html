Concept 01
The AK Consistency Hack
How a 1ms margin makes 23 adaptive keys reliable
When a combo times out or is invalidated (e.g., a non-member key is pressed), ZMK releases all buffered keys simultaneously. The &ak macro needs ak_tap_time (10ms) to tap the first key before activating the AK layer. Without help, the second key fires before the layer is ready — a race condition. The fix: a master combo on all 36 keys with timeout hold_key_event_delay (11ms), guaranteeing the AK layer activates 1ms before the second key fires.
Scenario: type C then P quickly
Without the Consistency Hack ✗ Race condition
Output —
With the Consistency Hack ✓ AK fires correctly
Output —
ak_tap_time (10ms) < hold_key_event_delay (11ms)
This 1ms margin is what makes the entire Adaptive Keys system reliable.

Concept 02
The Callum Mod Buffer
A 130ms safety net that makes mods forgiving
The htls thumb key (hold-preferred, resolves on interrupt or after action_lay_tapping_term = 90ms) activates the mod layer via &moAndBuffer. This macro presses &mo l_mod, waits for the thumb release, then keeps the layer alive for an extra 130ms buffer (action_mod_tapping_term). This means you don't have to think about timing — as long as the order is correct, it works. Note: the failure mode is about the layer disappearing — when the buffer expires, the mod layer deactivates and mod key positions revert to alpha keys. Already-armed sticky mods (like a Shift tapped via skm) remain active; the problem is that you can no longer reach new mod keys.

Concept 03
AK Keystroke Flow
The full adaptive key replacement pipeline
When you type C then P, the AK system detects the pair, activates l_akC, finds CP_to_TP at P's position, and fires REPLACE_CHAR_WITH_BIGRAM(T, P) — deleting C, typing T, then chaining into ak_P. The same-finger bigram CP becomes TP. Here is every step of that pipeline.
Screen —
Stage 1 — Key Press: C
Finger presses LM4 (left middle row, pinky). Binding: ak_C = &ak l_akC C
LM4 pressed
Stage 2 — &ak Macro Executes
The &ak macro (behavior-macro-two-param) fires with params l_akC and C.
&kp C (tap-ms: 10ms)
tap-ms: 10ms (key held then released)
&aksl l_akC (sticky layer ON)
Stage 3 — l_akC Layer Active
The AK layer is now active for my_ak_window (100ms). At position LT1 (where P lives on alpha), the binding is now CP_to_TP = &replace_char_with_bigram_TP.
l_akC active (100ms window)
LT1: CP_to_TP
Stage 4 — Key Press: P (hits l_akC)
At ~50ms, finger presses LT1. Instead of ak_P (alpha binding), it hits CP_to_TP on the active AK layer.
LT1 pressed
Resolves: &replace_char_with_bigram_TP
Stage 5 — REPLACE_CHAR_WITH_BIGRAM(T, P)
The replacement macro fires — a four-step sequence that transforms the output.
⌫ Backspace (deletes C)
&kp T (types T)
release LSHFT + RSHFT (safety)
chain: ak_P (enables next AK)
Stage 6 — Final Output
The computer received: C → Backspace → TP. The same-finger bigram CP has been eliminated. And because the chain used ak_P, the l_akP layer is now active — ready for the next adaptive key.

Concept 04
Linger Keys
The 100ms tap-vs-hold decision
Linger keys use ZMK's behavior-hold-tap with tap-preferred flavor and tapping-term-ms = linger_term (100ms). Release before 100ms = tap. Hold past 100ms = linger. The LK macro takes (TAP, LINGER) but internally reverses them to match ZMK's <HOLD, TAP> order.
Example: sqtDqt (RT4) — tap = ' single quote, linger = " double quote
0
press & hold
 
// The LK macro definition:
#define LK(NAME, TAP_BINDING, LINGER_BINDING) \
  bindings = <LINGER_BINDING>, <TAP_BINDING>;  // ← REVERSED!

// ZMK hold-tap expects: <HOLD_BINDING, TAP_BINDING>
// LK swaps them:        (TAP, LINGER) → <LINGER, TAP>

// Example: sqtDqt
LK(bk, &bk, &bk)
#define sqtDqt  &lk_bk DQT SQT  // tap=SQT('), linger=DQT(")

Concept 05
Combo Timeout Tiers
Tighter timeouts = fewer misfires
Three timeout tiers balance responsiveness against misfire risk. Adjacent keys (same hand, neighbors) get the tightest window (16ms) because accidental simultaneous presses are extremely unlikely. Non-adjacent keys on the same hand get 18ms (+2ms for the wider reach). Two-hand combos get 30ms — generous because opposite-hand coordination naturally varies more, and accidental simultaneous cross-hand presses are very rare.
